🌍 Splash Screen

Technical Description

The splash screen uses a full-viewport layout with CSS Flexbox and min-height: 100vh. The background uses the site's fixed gradient (linear-gradient(135deg, #1b7607, #000000)) inherited directly from style.css, overlaid with an animated SVG particle layer to add visual depth. The EcoFuture logo fades in via a CSS @keyframes animation (opacity 0→1 + translateY), followed by staggered reveals of the site tagline and a loading progress bar styled in the site's #4fa875 accent colour. JavaScript listens for the window.load event, then drives the progress bar to 100% before a transitionend callback redirects to index.html.

Accessibility

All animated elements respect the prefers-reduced-motion media query — users who have reduced motion enabled skip the animation and redirect immediately to index.html. The loading progress bar carries role="progressbar", aria-valuenow, and aria-valuemax="100" so screen readers announce loading progress. All text and background colour combinations on the dark gradient meet WCAG 2.1 AA contrast ratios.

⚡ SDG Action Impact Simulator

Technical Description

The simulator is an interactive single-page tool that lets users choose eco-friendly actions — switching to renewable energy, reducing meat consumption, planting trees — and instantly see the calculated CO₂ reduction impact. The page reuses the site's .btn and .btn-accent classes from style.css for all action buttons, and the #4fa875 / #31533e palette for the results panel. Interaction is driven by querySelectorAll and addEventListener('change') on checkboxes and <input type="range"> sliders. Impact values are stored in a JavaScript data object with research-based CO₂ coefficients. Results are displayed as an animated radial ring drawn on a <canvas> element, alongside a colour-coded summary bar. A reset button clears all selections and re-initialises the canvas context.

Accessibility

Every input has an explicit <label for=""> pairing. The canvas chart contains a fallback <table> inside it for screen reader users. The live score display uses role="status" and aria-live="polite" so score changes are announced without interrupting the user. All controls are fully keyboard operable — sliders respond to arrow keys and checkboxes to spacebar. The layout is verified at 200% zoom with no horizontal scroll.

🧩 Challenges and Lessons Learned

💡

The hardest challenge was synchronising the splash-screen animation timing to feel seamless across different devices and connection speeds. I learned to use transitionend events instead of hard-coded setTimeout delays for more reliable behaviour. Building the Canvas impact chart introduced me to the Canvas 2D API and the importance of providing accessible fallbacks. Maintaining visual consistency with the team's shared style.css — reusing the #4fa875 / #31533e palette, the .card layout, and .btn component — reinforced how valuable a shared design system is in a group project.

✅ Compliance

🛡️

All images on the splash screen include descriptive alt text. No third-party assets are used without an appropriate Creative Commons or royalty-free licence. The SDG 13 icon is sourced from the official UN SDG Assets package under open licence. Colour contrast ratios for all text have been verified against WCAG 2.1 AA. No personal user data is collected or stored on either page. JavaScript is vanilla with no external library dependencies, keeping load times minimal.
